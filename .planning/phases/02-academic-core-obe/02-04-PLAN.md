  <files>src/models/Exam.ts, src/models/ExamTimetable.ts, src/services/exam.service.ts, src/workers/exam.worker.ts, src/routes/exam.routes.ts, src/schemas/exam.schema.ts, tests/exam.service.test.ts</files>
  <behavior>
    - Create an Exam (Main/Supplementary) for a specific Academic Year/Semester.
    - Schedule ExamTimetable entries with (date, slot, roomId, courseId).
    - Prevent Room Clashes: Same (roomId, date, slot) cannot have two exams.
    - Detect Student Clashes: A student registered for two courses (from CourseRegistration) cannot have exams in the same date/slot.
    - Check Eligibility: Student must have >= 75% attendance (from AttendanceRecord) to generate a Hall Ticket.
    - Generate Hall Ticket PDF: Use BullMQ to queue a job that uses `pdfmake` to generate a PDF and store/return its URL.
  </behavior>
  <action>

    2. Define `ExamTimetable` model: { tenantId, examId, courseId, roomId, date, slot }. Add unique index on { tenantId, roomId, date, slot }.
    3. Implement `exam.service.ts`:
       - `scheduleExam`: Validates room availability and checks for registered student clashes.
       - `requestHallTicket`: Checks attendance eligibility and pushes job to `hall-ticket-gen` queue.
    4. Implement `src/workers/exam.worker.ts`:
       - Consumer for `hall-ticket-gen`.
       - Uses `pdfmake` to build a layout with student details and their personal exam timetable.
       - Saves PDF (mock storage or MinIO if available from Phase 1).
    5. Write tests to verify clash detection and eligibility logic.
  </action>

    - SGPA Calculation: (Sum of (GradePoint * Credits)) / (Sum of Credits) for a semester.

  </behavior>

       - `AbsoluteGradingStrategy` implementation (e.g., >90=O, >80=A+).
