  <name>Task 2: Circulation & Fine Calculation (LIB-02)</name>
  <files>src/models/Circulation.ts, src/services/lib.service.ts, tests/lib.service.test.ts</files>
  <behavior>
    - `issueBook(bookId, userId)`:
      - Validates `Book.availableCopies > 0`.
      - Creates `Circulation` record with `status: 'Issued'`.
      - Decrements `Book.availableCopies`.
      - Returns the `Circulation` object.
    - `returnBook(circulationId)`:
      - Updates `Circulation`: `returnDate = now()`, `status: 'Returned'`.
      - If `returnDate > dueDate`, calculates `fineAmount` based on days overdue (daily rate from tenant config or constant).
      - Increments `Book.availableCopies`.
      - Triggers `processNextReservation(bookId)`.
  </behavior>
  <action>
    - Define `Circulation` schema: `bookId`, `userId`, `issueDate`, `dueDate`, `returnDate`, `status` (Issued, Returned), `fineAmount`. Apply `mongooseIsolation`.
    - Implement issue and return logic in `lib.service.ts`.
    - Implement fine calculation logic.
  </action>
  <verify>
    <automated>npm test tests/lib.service.test.ts</automated>
  </verify>
  <done>Book circulation and manual fine calculation on return are verified (LIB-02).</done>
</task>
