<task type="auto">
  <name>Task 1: Expand AnalyticsService with core KPIs</name>
  <files>src/services/analytics.service.ts</files>
  <action>
    Modify `AnalyticsService` to include the following aggregation-based methods:
    1. `getEnrollmentAnalytics(tenantId, filters)`: Aggregates student counts grouped by department, academic year, and gender.
    2. `getFeeAnalytics(tenantId, filters)`: Aggregates total demand vs. paid amount, grouped by fee category and student batch.
    3. `getAcademicAnalytics(tenantId, filters)`: Calculates average CGPA/SGPA, course-wise pass percentages, and attendance-performance correlation.
    
    Performance Note: For high-frequency KPIs (like daily attendance trends), implement a basic "Computed Pattern" (caching results in a dedicated `AnalyticsCache` collection) to avoid re-aggregating millions of records on every request.
    
    Ensure all pipelines include `{ $match: { tenantId } }` as the first stage for isolation.
    Use `mongoose.Types.ObjectId` for ID fields in aggregations.
  </action>
  <verify>
    <automated>npm test tests/analytics.service.test.ts</automated>
  </verify>
  <done>AnalyticsService has comprehensive KPI methods returning validated data structures.</done>
</task>
